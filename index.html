Variables and data types

Variables

A way to store information

Declaration

// value here can be changed
let info; 
// value in here can't be changed
const info2 = 'JavaScript rules!';

Assign value

// all good
info = 'JavaScript rules!'; 
 // ERROR due to being constant
info2 = 'CSS@WIT rules!';

Data Types

Strings const bananas = "I have some bananas"; Contained in quotes, quotes must match (single ' or double "). Escape quotes with \
Numbers let numberOfBananas = 5; Only ONE number type -- all numbers act like numbers
Boolean let haveBananas = true; true or false, no quotes!
Undefined let strawberries; No assigned value
Null const pumpkins = null; An intentionally empty value (not the same as 0 or undefined)

Arrays

A list of variables

let list = [];
let animals = ['cat','dog','bird'];
console.log(animals); // this will print ['cat', 'dog', 'bird']
console.log(animals[0]); // this will print 'cat'
animals[0] = 'lemur';
console.log(animals[0]); // this will print 'lemur'
console.log(animals.length); // this will print 3
animals.push('ferret');
console.log(animals); // this will print ['cat', 'dog', 'bird', 'ferret']
animals.pop();
console.log(animals); // this will print ['cat', 'dog', 'bird']

Notes on Variables

  • JavaScript variables are "loosely typed"
  • Names begin with a letter, _, or $
  • Contain letters, numbers, _, and $
  • *don't use let when you can use const

Functions, scope, and more

Structure

// Function keyword and declaration
function myFunction() {
  alert('hello');
}
// Function expression
let myFunction = () => {
  alert('hello');
}
// Call it
myFunction();

Scope

let x = 1;
function makeY(number) {
    const y = number;
    return y;
}
function changeX() {
    x = x * 2;
}

Closures

Scope of an inner function includes the outer.

function addSquares(a,b) {
   function square(x) {
      return x * x;
   }
   return square(a) + square(b);
}
function setAlarm(message, timeout) {
  function handle() {
    console.log(message);
  }
  setTimeout(handle, timeout);
}
setAlarm("Wake up!", 100);

Self-executing functions

((() => {
  const foo = 'Hello';
  const bar = 'World!';

  function baz(){
      return `${foo} ${bar}`; // equivalence to (foo + " " + bar)
  }
}))();

Class

Create a template for objects to follow (I honestly don't know how to explain better)

class Animal {
    constructor(type='Unspecified', sound='silent') {
        this.type = type,
        this.sound = sound
    }

    setName(newName) {
        this.name = newName
    }
}
let leopard = new Animal('cat', 'grr');
console.log(leopard.type); // print 'cat'
console.log(leopard.sound); // print 'grr'

Extra: ES6 vs its predecessor (ES5)

For those who have seen JavaScript code before

This is Equivalence

ES5

var name = "Jackson Schwarz";
var x;
//saying hello
console.log("Hello " + name);
[1, 2, 3].map(function(n) { return n * 2; }, this);
// -> [ 2, 4, 6 ]

ES6 (Cooler)

const name = "Jackson Schwarz";
let x;
//saying hello
console.log(`Hello ${name}`);
[1, 2, 3].map(n => n * 2, this);
// -> [ 2, 4, 6 ]

Why ES6?

  • Cleaner and much more convinient
  • (#Advanced)Closer to Object-oriented Programming, which means it will look similar to Java and C++
